# GitOps deployment with ArgoCD (OpenShift GitOps)

> ⚠️ **AI-generated & untested.** Generated by Claude; not yet deployed or tested
> on a real OpenShift/ArgoCD cluster (only `kustomize build` + offline YAML
> checks). Review and validate in a non-production environment first. See the
> [root README](../README.md) disclaimer.

This directory performs the **declarative** deployment with ArgoCD, using the
**App-of-Apps** pattern. It is the recommended path. (The imperative
`install.sh` at the repo root is still available for clusters without ArgoCD.)

```
argocd/project.yaml             AppProject 'traefik' (scopes repos/dest/resources)
argocd/root-app.yaml            App-of-Apps root  → points to argocd/apps/
argocd/apps/traefik.yaml        Application (Helm) → chart 41.0.2 + helm/values-traefik.yaml   [wave 0]
argocd/apps/traefik-dashboard.yaml  Application (Kustomize) → manifests/                       [wave 1]
```

Order guaranteed by **sync-waves**: `traefik` (wave 0) installs the `traefik.io`
CRDs and creates/labels the namespace; `traefik-dashboard` (wave 1) deploys
oauth2-proxy, middlewares and IngressRoutes that depend on those CRDs.

## 0. Prerequisites

- **OpenShift GitOps Operator** installed (provides ArgoCD). Usual namespace:
  `openshift-gitops`. With vanilla ArgoCD it's typically `argocd`.
  Requires **ArgoCD ≥ 2.6** (OpenShift GitOps ≥ 1.8) for *multi-source* and
  `managedNamespaceMetadata`.
- The ArgoCD instance allowed to deploy into the `traefik` namespace.
- Your code in a git repo ArgoCD can reach.

## 1. Adjust the placeholders

The `repoURL` is already set to
`https://github.com/nubenetes/traefik-keycloak-openshift-gitops.git`. In the
**4** files under `argocd/` review:

| Placeholder | To |
|---|---|
| `targetRevision: main` | your branch or tag |
| `namespace: argocd` (metadata) | your ArgoCD namespace (e.g. `openshift-gitops`) |

Quick pass (Git Bash / Linux / macOS):

```bash
ARGO_NS=openshift-gitops           # or 'argocd'
sed -i "s/^  namespace: argocd .*/  namespace: $ARGO_NS/" argocd/root-app.yaml argocd/project.yaml argocd/apps/*.yaml
```

And in `helm/values-traefik.yaml` / `manifests/` replace the dummy domains
(`traefik.apps.example.com`, `keycloak.apps.example.com`, `myrealm`) — see the
main README §4.

## 2. Create the secrets (out-of-band, once)

ArgoCD does **not** manage the secrets (they are not in git). Create them by hand:

```bash
# oauth2-proxy (Keycloak client secret + cookie secret)
cp secrets/oauth2-proxy-secret.example.yaml secrets/oauth2-proxy-secret.yaml
# edit real values; then:
oc apply -f secrets/oauth2-proxy-secret.yaml

# Dashboard TLS (see docs/tls-secret.md; with cert-manager it's declarative)
oc create secret tls traefik-dashboard-tls -n traefik --cert=tls.crt --key=tls.key
```

> The `traefik` namespace is created by the `traefik` Application on its first
> sync. If you need to apply the secrets BEFORE that, create it yourself:
> `oc create namespace traefik`.

## 3. Bootstrap (once)

```bash
# 1) Dedicated AppProject (must exist BEFORE the Applications that use it)
oc apply -n openshift-gitops -f argocd/project.yaml      # adjust the namespace
# 2) App-of-Apps root
oc apply -n openshift-gitops -f argocd/root-app.yaml
```

ArgoCD detects the two child Applications (in the `traefik` AppProject) and syncs
them in order. Follow progress in the ArgoCD UI or with the CLI:

```bash
argocd app list
argocd app get traefik
argocd app get traefik-dashboard
```

## 4. Operations

- **Every change goes through git**: edit `helm/values-traefik.yaml` or
  `manifests/…`, commit/push and ArgoCD applies it on its own (auto-sync +
  self-heal enabled).
- **Bump the Traefik version**: change `targetRevision: 41.0.2` in
  `argocd/apps/traefik.yaml` (see `helm search repo traefik/traefik --versions`),
  commit and push. Keep Traefik **≥ v3.4**.
- **Update secrets**: reapply the Secret and restart oauth2-proxy
  (`oc rollout restart deploy/oauth2-proxy -n traefik`); changing a Secret does
  not restart the pod on its own.
- **Uninstall**: delete the root Application (the finalizer cascade-deletes the
  children and their resources):
  `oc delete -n openshift-gitops application/traefik-root`.
  Then clean up external state (secrets, DNS, Keycloak client, CRDs — see main
  README §7).

## Notes

- `ServerSideApply=true` on the Traefik app avoids the *"metadata.annotations:
  Too long"* error with its large CRDs.
- The namespace is created and labeled
  (`pod-security.kubernetes.io/enforce: restricted`) via
  `managedNamespaceMetadata` on the `traefik` Application — no standalone
  Namespace manifest.
- The child Applications use the **dedicated AppProject `traefik`**
  (`argocd/project.yaml`): it scopes `sourceRepos`, `destinations` and the
  allowed cluster-scoped resources (Namespace, CRDs, ClusterRole/Binding,
  IngressClass).
- **Secret in git (optional):** instead of the out-of-band `oc apply`, seal the
  Secret with `kubeseal` (see `manifests/oauth2-proxy-sealedsecret.example.yaml`)
  and enable it in `manifests/kustomization.yaml`.
